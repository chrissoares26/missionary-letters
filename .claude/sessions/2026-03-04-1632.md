### Problems Encountered and Solutions

**1. Pinia Colada mutation `status` initializes as `'pending'`**
- Symptom: Submit button showed "Salvando..." disabled on page load
- Root cause: Pinia Colada v0.21.7 `useMutation` initializes `state.status = 'pending'` (meaning "no data yet"), not `'idle'`
- Fix: Use `isLoading` from `useMutation` return (backed by `asyncStatus === 'loading'`), which correctly starts as `false`
- Pattern: `const { mutateAsync, isLoading } = useMutation(...)` — never use `status === 'pending'` for active-submission detection

**2. Edge Function CORS blocked by `verify_jwt: true`**
- Symptom: Console CORS error on OPTIONS preflight; embedding never triggered
- Root cause: Supabase's gateway-level JWT verification intercepts OPTIONS requests before function code runs, returning 401 with no CORS headers
- Fix: Redeploy with `verify_jwt: false` + add `CORS_HEADERS` constant + handle `OPTIONS` at top of function before auth check
- Rule: Any Edge Function that handles auth manually must use `verify_jwt: false`

**3. `style_emails` table missing from Supabase Realtime publication**
- Symptom: Badges never flipped from "Processando..." to "Pronto ✓" even after embedding was persisted
- Root cause: `style_emails` was not in `supabase_realtime` publication — Postgres changes not broadcast
- Fix: Migration `enable_realtime_style_emails` — `ALTER PUBLICATION supabase_realtime ADD TABLE public.style_emails`
- Note: Any new table requiring `postgres_changes` subscriptions needs this migration

---

### Breaking Changes / Important Findings

- **Pinia Colada `useMutation` status semantics differ from `useQuery`**: mutations start as `'pending'` (no data), not `'idle'`. Always use `isLoading` for "is submitting" detection.
- **Supabase Realtime requires explicit publication enrollment** per table — it is NOT automatic for new tables.
- **Edge Functions with manual auth must use `verify_jwt: false`** — the gateway JWT check does not pass CORS headers on 401, breaking browser preflight.

### Tips for Future Developers

1. **Pinia Colada mutations**: use `isLoading` for "in-flight" detection, `status` for success/error state after completion
2. **New Supabase tables with Realtime**: always add `ALTER PUBLICATION supabase_realtime ADD TABLE public.<table>` migration
3. **Edge Functions with manual auth**: always use `verify_jwt: false` and handle OPTIONS preflight explicitly
4. **Embedding badge flow**: `StyleEmailList` seeds `localEmbeddings` from props on mount, Realtime updates it. `EmbeddingStatusBadge` has a 60s timer — error state is purely client-side; re-adding the email will retry
5. **`triggerEmbedding` is fire-and-forget**: errors are logged as warnings only. If a badge shows "Erro", the email exists in DB without an embedding — delete and re-add to retry
6. **Next session** should tackle Epic 4 Stories 4.3 + 4.4 (`style_match` function + profile synthesis), then move to Epic 5 (Campaign Draft Generation)
