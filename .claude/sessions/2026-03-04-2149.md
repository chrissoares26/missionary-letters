### Breaking Changes / Important Findings

#### **RAG Pipeline Architecture**
1. **Style ingestion** → user adds emails via modal form
2. **Embedding generation** → automatic via `style_embed_upsert` Edge Function (fire-and-forget)
3. **Profile synthesis** → on-demand via `style_profile_generate` (requires min 3 emails)
4. **Similarity retrieval** → `style_match` Edge Function retrieves top-K examples for campaign generation
5. → **Campaign generation** (Epic 5) will use profile + matched examples

#### **Pinia Colada Patterns Learned**
- `useMutation` `status` starts as `'pending'` (no data yet), not `'idle'`
- Always use `isLoading` for "is submitting" detection, not `status === 'pending'`
- `useQuery` with `.maybeSingle()` prevents 406 errors for optional data

#### **Supabase Edge Function Patterns**
- **Dual-client pattern required for manual auth:**
  - `userClient` (anon + JWT) for auth verification
  - `adminClient` (service role) for DB writes
- **CORS headers must be explicit** when `verify_jwt: false`
- **Always handle OPTIONS preflight** before auth checks

#### **Vector Search with pgvector**
- Cosine similarity via `<=>` operator: `1 - (embedding <=> query_embedding)` gives similarity score (0-1)
- RPC functions allow complex queries not supported by Supabase JS client directly
- Index on vector column improves performance: `CREATE INDEX ON style_emails USING ivfflat (embedding vector_cosine_ops)`

### Tips for Future Developers

#### **Working with Style Profile**
1. Profile generation requires min 3 emails with embeddings
2. Profile is regenerated on demand via "Regenerar" button (doesn't auto-update when new emails added)
3. Profile analyzes max 20 most recent emails to stay within OpenAI token limits
4. `profile_json` is jsonb — always use optional chaining when accessing nested properties

#### **Vector Similarity Search**
1. `match_style_emails()` RPC function is the gateway to vector search
2. Function returns results sorted by similarity (highest first)
3. Similarity score: 1 - cosine_distance (higher = more similar)
4. Filter by `owner_id` enforced in RPC to prevent cross-user data leaks
5. Min 3 emails with embeddings required before similarity search works

#### **Modal Form Pattern**
1. Modal uses Teleport to `body` for proper z-index stacking
2. Backdrop click closes modal via `@click.self`
3. Mobile uses slide-up animation, desktop uses scale
4. Safe-area padding required for iOS notch: `padding-bottom: env(safe-area-inset-bottom, 0px)`

#### **Collapsible List Pattern**
1. `displayedEmails` computed slices based on `isExpanded` state
2. Button only shows when `emails.length > PREVIEW_LIMIT`
3. Button text dynamically shows total count: "Ver todos os X emails"
