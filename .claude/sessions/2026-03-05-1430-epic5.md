**Issues Encountered & Solutions**

1. **Pinia Colada mutation status**
   - Problem: `status.value === 'pending'` is true on initial state — buttons disabled on page load
   - Fix: Use `isLoading.value` (true only while actively running)

2. **Edge Function CORS**
   - Problem: CORS headers only on OPTIONS preflight, not on POST responses
   - Fix: Added `corsHeaders` constant spread into ALL response objects (v2)

3. **Inter-function auth (style_match 401)**
   - Problem: `draft_generate` called `style_match` via `adminClient.functions.invoke()` which attaches service role key — `style_match`'s `auth.getUser()` can't resolve a user from it
   - Fix: Forwarded original user JWT via `headers: { Authorization: authHeader }` in functions.invoke (v3)

4. **Wrong parameter name**
   - Problem: Sent `top_k: 6` to `style_match` which expects `k`
   - Fix: Changed to `k: 6` (v3)

5. **Realtime not updating UI**
   - Problem: `campaign_content` table not in `supabase_realtime` publication
   - Fix: `ALTER PUBLICATION supabase_realtime ADD TABLE campaign_content;` via migration

6. **CSS variables undefined across all campaign components**
   - Problem: Components used `--primary`, `--border`, `--background`, `--background-secondary`, `--primary-light` — none defined in `tokens.css`
   - Actual token names: `--action-primary`, `--border-default`, `--bg-canvas`, `--bg-muted`, `--bg-surface`
   - Fix: Global replace across all 6 affected files

| Wrong (undefined) | Correct |
|---|---|
| `--primary` | `--action-primary` (#d88939 amber) |
| `--border` | `--border-default` (#d2c7b2) |
| `--background` | `--bg-canvas` (#f9f6ef) |

| `--primary-light` | `--bg-muted` (#f2ebdd) |

**Tip for future work:** Any new component that uses color/border tokens must use the names from `src/styles/tokens.css` `:root` block — not generic names like `--primary`.

### Supabase Notes

- **`draft_generate`** deployed as v3 — v1 had no CORS, v2 fixed CORS, v3 fixed auth + param name
- **Realtime publication:** Only `style_emails` was originally in `supabase_realtime`. `campaign_content` was added via migration. Future tables that need Realtime must be explicitly added.
- **Inter-function calls:** When one Edge Function calls another via `adminClient.functions.invoke()`, the admin client's service role key is attached. If the target function calls `auth.getUser()`, it will 401. Always forward the original user JWT via `headers: { Authorization: authHeader }`.
