#### Tips for Future Developers

- **`missionaries_autodeactivate` auth:** uses `Authorization: Bearer <SUPABASE_SERVICE_ROLE_KEY>` — not a user JWT. The cron trigger sends this automatically; manual invocations must include it.
- **Rate limiting in `campaign_send`:** only kicks in for lists >15 recipients (`shouldRateLimit` flag). Jitter is 300–500ms per send.
- **`campaign_recipients` has no `owner_id`** — ownership enforced via join to `campaigns.owner_id`. RLS policies rely on this join path.
- **All invoke-called Edge Functions** (`draft_generate`, `style_embed_upsert`, `campaign_send`, `oauth_google_callback`) deployed with `--no-verify-jwt`; they perform explicit `auth.getUser()` internally.
- **Error taxonomy** in `documentation/error_taxonomy.md` is the reference for all new error handling — follow the class + recovery pattern when adding new flows.
